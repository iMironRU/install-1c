# Инструкции для Claude Code

Это техническая книга. Перед любым действием прочитай контекст:

- **`spec/constitution.md`** — принципы книги: что она ЕСТЬ и что НЕ ЕСТЬ.
  Если предложение выходит за рамки конституции — откажись от него,
  даже если оно кажется полезным.
- **`spec/specification.md`** — для кого книга, что читатель знает на входе,
  метрики успеха, ограничения.
- **`spec/decisions.md`** — лог принятых решений. Перед предложением нового
  решения проверь, не принималось ли оно уже.
- **`docs/style-guide.md`** — требования к стилю параграфа.
- **`metadata.yaml`** — версия, форматы, статусы.

---

## Структура репозитория

```
chapters/
  NN_название_модуля/
    NN-NN_название_параграфа.md   ← frontmatter обязателен (см. ниже)

assets/img/                       ← обложка и иллюстрации
spec/                             ← конституция, спецификация, решения
docs/                             ← гайды для автора
```

### Именование файлов

Модули: `00_vvedenie`, `01_leksika`, `02_semantika`
Параграфы: `00-01_chto_takoe.md`, `00-02_ot_dannyh.md`

Нумерация непрерывная внутри модуля, с ведущим нулём.

### Frontmatter параграфа (обязательный)

```yaml
---
status: draft      # draft | review | ready
---
```

- `draft` — написан, не финализирован
- `review` — на проверке
- `ready` — финальный, идёт в релиз

## Команды (slash-команды Claude Code)

### `/book-write §N.N [тема или краткое описание]`

Написать или продолжить параграф. Алгоритм:

1. Прочитай `spec/constitution.md` и `spec/specification.md`.
2. Найди соответствующий файл в `chapters/` или создай его по шаблону.
3. Убедись, что тема параграфа не выходит за рамки конституции.
4. Напиши текст в стиле, описанном в `docs/style-guide.md`.
5. Поставь `status: draft` в frontmatter.
6. В конце кратко объясни: какие решения принял и почему.

### `/book-analyze §N.N`

Проверить параграф. Проверяй по порядку:

1. **Конституция** — нет ли отклонений от принципов книги?
2. **Спецификация** — соответствует ли уровень подготовки читателя?
3. **Стиль** — выполнены ли требования из `docs/style-guide.md`?
4. **Консистентность** — не противоречит ли соседним параграфам?
5. **Код** (если есть) — работает ли пример, нет ли ошибок?

Выдай результат в формате:
```
✅ Конституция: ...
✅ Спецификация: ...
⚠️  Стиль: [конкретное замечание]
✅ Консистентность: ...
```

### `/book-plan модуль-N`

Спланировать или уточнить модуль. Сделай:

1. Прочитай `spec/specification.md` — что читатель должен уметь после книги.
2. Посмотри существующие параграфы модуля в `chapters/`.
3. Предложи список параграфов с 1-2 предложениями описания каждого.
4. Выяви неоднозначности: задай не более 3 уточняющих вопросов.
5. Зафикси принятые решения в `spec/decisions.md`.

## Что не делать

- Не добавляй темы, которые противоречат конституции, даже если они «полезны».
- Не меняй `status: ready` на что-то другое без явного запроса автора.
- Не изменяй `metadata.yaml` напрямую — только через `./book.sh`.
- Не генерируй длинные тексты без запроса — спроси план сначала.
