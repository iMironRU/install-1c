Проверь параграф книги по всем критериям качества.

Аргументы: $ARGUMENTS
Формат: `§N.N`
Пример: `§1.2`

## Алгоритм

1. **Найди файл параграфа** в `chapters/` по номеру из аргумента

2. **Прочитай эталоны** для проверки:
   - `spec/constitution.md` — принципы книги
   - `spec/specification.md` — целевой читатель и его уровень
   - `docs/style-guide.md` — требования к стилю

3. **Проверь по пяти критериям** (строго в этом порядке):

### 1. Конституция
- Тема параграфа входит в область, описанную в `spec/constitution.md`?
- Нет ли утверждений, противоречащих принципам книги?
- Не включает ли параграф то, что конституция явно исключает?

### 2. Спецификация
- Соответствует ли уровень изложения читателю из `spec/specification.md`?
- Не объясняет ли параграф то, что читатель уже знает (по условиям spec)?
- Не требует ли параграф знаний, которых у читателя нет?

### 3. Стиль
- Выполнена ли структура из `docs/style-guide.md` (Что → Как → Итог)?
- Соблюдён ли тон и запреты из style-guide?
- Нет ли нарушений из раздела «Что не делать»?

### 4. Консистентность
- Прочитай соседние параграфы (предыдущий и следующий в `chapters/`)
- Нет ли противоречий с соседними параграфами?
- Нет ли повторов — темы, которая уже раскрыта в другом параграфе?

### 5. Код (если есть в параграфе)
- Каждый пример компилируется / запускается без ошибок?
- Нет ли магических чисел без объяснения?
- Комментарии объясняют «почему», а не «что»?

## Формат вывода

```
✅ Конституция: соответствует / ⚠️  Конституция: [конкретное замечание]
✅ Спецификация: соответствует / ⚠️  Спецификация: [конкретное замечание]
✅ Стиль: соответствует / ⚠️  Стиль: [конкретное замечание]
✅ Консистентность: соответствует / ⚠️  Консистентность: [конкретное замечание]
✅ Код: нет кода / ✅ Код: рабочий / ⚠️  Код: [конкретная ошибка]

Итог: [1-2 предложения — готов ли параграф к переводу в review]
```

## Важно

- Каждое замечание — конкретное и actionable, не общее («улучшить стиль»)
- Если spec/ не заполнен — укажи это как блокер: проверка по критериям 1 и 2 невозможна
- Не меняй статус параграфа — только рекомендуй
