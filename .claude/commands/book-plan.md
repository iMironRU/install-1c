Спланируй или уточни состав модуля книги.

Аргументы: $ARGUMENTS
Формат: `модуль-N [необязательное описание задачи]`
Пример: `модуль-1` или `модуль-2 нужно добавить раздел про регистры`

## Алгоритм

1. **Прочитай контекст книги**:
   - `spec/specification.md` — что читатель должен уметь ПОСЛЕ всей книги
   - `spec/constitution.md` — принципы и ограничения
   - `spec/decisions.md` — что уже решено, чтобы не предлагать повторно

2. **Изучи текущее состояние модуля**:
   - Найди папку модуля в `chapters/NN_название/`
   - Прочитай все существующие параграфы
   - Запусти `./book.sh status` (или посмотри frontmatter файлов) — какие параграфы готовы, какие в черновике

3. **Сформируй предложение** по составу модуля:
   - Список параграфов с номерами (§N.1, §N.2, ...)
   - Для каждого — 1-2 предложения: что параграф даёт читателю
   - Укажи какие уже существуют, какие нужно создать
   - Логика: каждый параграф — одна идея, один навык

4. **Выяви неоднозначности** — задай не более 3 вопросов автору:
   - Только если действительно нужен ответ для планирования
   - Конкретные вопросы, не общие («что важнее?»)

5. **Зафикси принятые решения** в `spec/decisions.md`:
   - Только если в ходе планирования были приняты нетривиальные решения
   - Формат: дата, проблема, решение, почему, альтернативы
   - Не добавляй очевидные решения — только то, что неочевидно и может быть оспорено позже

## Формат вывода

```
## Модуль N: [название]

**Цель модуля:** что читатель умеет после прохождения этого модуля

**Текущее состояние:** N параграфов (X готовы, Y в черновике, Z нужно создать)

### Предлагаемый состав

| § | Название | Статус | Что даёт читателю |
|---|----------|--------|-------------------|
| §N.1 | ... | существует / создать | ... |
| §N.2 | ... | существует / создать | ... |

### Вопросы (если есть)

1. ...
2. ...
```

## Важно

- Не предлагай параграфы, выходящие за рамки конституции
- Проверь `spec/decisions.md` — не предлагай то, что уже было отвергнуто
- Не планируй больше 7-8 параграфов в одном модуле — это признак что модуль надо разбить
