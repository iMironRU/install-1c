## [2026-06-09] Целевая версия платформы — 8.5 как база, 8.3 как ветка

**Проблема:** на момент написания актуальна 8.5, но большой парк инсталляций на 8.3. Без явного решения шаги начнут расползаться по версиям.

**Решение:** базовая целевая версия — 8.5. Там, где различия с 8.3 существенны (установщик, утилиты сервера, способ публикации), в шаге появляется `platform_version: "8.3"` и альтернативные секции action/postconditions. 8.2 и старше — вне scope.

**Почему:** 8.5 — текущая, направление развития. Делать базой устаревающую версию — против движения системы.

**Альтернативы:** только 8.5 (отброшено: парк 8.3 релевантен аудитории); 8.3 как база (отброшено: устаревает); полная матрица версий (отброшено: комбинаторный взрыв).

## [2026-06-09] Структура книги — по сценариям, не по ОС

**Проблема:** материал можно организовать по ОС (Windows / Linux / macOS) или по сценарию установки (dev-learn / thin-client / ...).

**Решение:** главы организованы по сценариям. Внутри каждой главы — разделы по ОС там, где различия существенны.

**Почему:** читатель приходит с задачей («хочу учиться разрабатывать»), а не с выбором ОС. Навигация от сценария естественнее.

**Альтернативы:** по ОС (отброшено: читатель вынужден читать три главы, чтобы найти свою задачу).

## [2026-06-09] Нет единого сквозного примера

**Проблема:** нужен ли сквозной пример, который идёт через всю книгу?

**Решение:** нет. Каждый сценарий самодостаточен. Переиспользуемые шаги (например, скачивание дистрибутива) вынесены в `src/steps/` и ссылаются между сценариями.

**Почему:** читатель приходит за конкретным сценарием, не за всей книгой. Сквозной пример создаёт искусственную связность там, где её нет.
