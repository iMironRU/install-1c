# Конституция книги

---

## Что эта книга ЕСТЬ

- Пошаговое руководство по установке 1С:Предприятие 8.5 для пяти чётко определённых сценариев: обучение разработке, тонкий клиент, файловый офис, серверная установка, облачный 1С:Фреш.
- Инструкции для трёх операционных систем (Windows, Linux, macOS) — там, где сценарий применим.
- Проверяемые инструкции: каждый шаг содержит критерий успеха, не «должно открыться», а конкретная команда и ожидаемый результат.
- Три уровня изложения для одного действия: язык пользователя, язык бизнес-задачи, точные команды и пути.

## Что эта книга НЕ ЕСТЬ

- Не учебник по разработке на 1С — только установка и настройка среды.
- Не руководство по администрированию 1С после установки — до первого входа.
- Не инструкция по использованию неофициальных дистрибутивов. Только `users.v8.1c.ru` и `1c.ru`.
- Не универсальный скрипт «поставить всё». Каждый сценарий имеет явную границу.
- Не покрывает 8.2 и старше. 8.3 — отдельные ветки там, где различия существенны.

## Метафора книги

Книга — как опытный коллега, который уже сделал это десять раз и точно знает, где споткнуться. Не читает лекцию, а ведёт за руку: «делай вот это, потом проверяй вот так».

## Принципы изложения

1. **Цель до действий.** Каждый сценарий начинается с «зачем» и критерия «установлено».
2. **Постусловие исполняемо.** «Откройте окно» — не проверка. «Команда возвращает 0» — проверка.
3. **Три языка в одном шаге.** Что видит пользователь, что происходит в системе, что делает машина.
4. **Лицензирование — отдельный поток.** Установка платформы и активация лицензии не смешиваются.
5. **Только официальные источники.** Альтернативные сборки упоминаются только как «сюда не идём — вот почему».

## Что автор не будет делать никогда

- Не смешивать установку платформы и активацию лицензии в одном шаге.
- Не писать «должно работать» без проверяемого постусловия.
- Не ссылаться на неофициальные источники дистрибутивов в основном тексте.
- Не добавлять сценарии без целевого состояния и критерия проверки.

## Аудитория (кратко)

Читатель: системный администратор / IT-специалист, бухгалтер или рядовой пользователь 1С, начинающий разработчик — в зависимости от сценария.

Читатель НЕ: опытный 1С-администратор, которому не нужны пошаговые инструкции.
