# ADR-0001: Целевая версия платформы

**Статус:** принято
**Дата:** 2026-06-09
**Авторы:** iMironRU

## Контекст

На момент создания репо актуальная версия — 1С:Предприятие 8.5.
В обиходе остаётся большой парк инсталляций на 8.3.
8.2 и старше — в продакшене встречаются, но новые установки не производятся.

Без явного решения о scope шаги начнут расползаться: одни авторы будут писать под 8.5, другие — под 8.3, у пользователя возникнет ощущение раздолбанной документации.

## Решение

1. **Базовая целевая версия — 8.5.** Шаги пишутся под неё. Команды, пути, ссылки на дистрибутивы — для 8.5.
2. **8.3 — отдельные ветки.** Там, где различия существенны (изменения установщика, утилит сервера, способа публикации), в шаге появляется параметр `platform_version: "8.3"` и альтернативные секции `action.*` / `postconditions.*`.
3. **8.2 и старше — вне scope.** Не упоминаются в `action`. Допустимы упоминания в `didactic_notes.pitfalls` исключительно как контекст («раньше делалось так, сейчас иначе»).

## Последствия

- В `src/steps/<id>.yaml` команды по умолчанию относятся к 8.5.
- Для шагов с расходящимися ветками 8.5 / 8.3 используется поле `platform_version` и парные блоки `action` / `postconditions`.
- В дистрибутивах и URL в шагах используется актуальная для 8.5 структура.
- При выходе 8.6 — новый ADR с переходом базовой версии, без переписывания всех шагов сразу: парные ветки 8.6 / 8.5 на переходный период.

## Альтернативы, которые рассматривались

- **Только 8.5.** Отброшено: большой парк 8.3 остаётся релевантным аудитории «Школы инженеров 1С».
- **8.3 как база, 8.5 как ветка.** Отброшено: 8.5 — текущая, направление развития платформы. Делать базой устаревающую версию — против движения системы.
- **Полная матрица версий.** Отброшено: комбинаторный взрыв и обесценивание автоматических тестов.
