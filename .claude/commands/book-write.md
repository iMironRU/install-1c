Напиши или продолжи параграф книги.

Аргументы: $ARGUMENTS
Формат: `§N.N [тема или краткое описание]`
Пример: `§1.2 что такое метаданные`

## Алгоритм

1. **Прочитай контекст книги** — обязательно перед написанием:
   - `spec/constitution.md` — что книга ЕСТЬ и что НЕ ЕСТЬ, принципы изложения
   - `spec/specification.md` — для кого книга, что читатель знает на входе
   - `docs/style-guide.md` — требования к стилю: тон, структура, что не делать

2. **Найди или создай файл параграфа:**
   - Определи номер модуля и параграфа из аргумента (например `§1.2` → модуль 01, параграф 02)
   - Найди соответствующий файл в `chapters/` по номеру
   - Если файл не существует — создай его по шаблону (см. ниже)
   - Именование: `chapters/NN_название_модуля/NN-NN_название_параграфа.md`

3. **Проверь соответствие конституции:**
   - Тема параграфа не выходит за границы, описанные в `spec/constitution.md`
   - Если тема противоречит конституции — сообщи об этом автору до написания

4. **Напиши текст** в соответствии со стилем из `docs/style-guide.md`:
   - Структура: Что (зачем это читателю) → Как (материал с примерами) → Итог
   - Тон и ограничения — из style-guide.md
   - Если в параграфе есть код — он должен быть рабочим

5. **Frontmatter** — обязательно в начале файла:
   ```yaml
   ---
   status: draft
   ---
   ```

6. **После написания** — кратко (2-4 предложения) объясни:
   - Какие решения принял при написании
   - Что намеренно упростил или оставил за кадром
   - Если есть TODO/FIXME — перечисли их

## Шаблон нового файла

```markdown
---
status: draft
---

# Название параграфа

> TODO: написать параграф §N.N — тема

```

## Важно

- Не меняй `status: ready` без явного запроса автора
- Не добавляй темы, которые противоречат конституции
- Не генерируй длинный текст без понимания контекста — если spec/ не заполнен, сообщи автору
- Проверь `spec/decisions.md` — возможно по этой теме уже было принято решение
